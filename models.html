models module

Database access layer for the Flask Blog application.

Provides SQLite-backed CRUD operations for blog posts, including pagination, search, and statistics queries.

models.get_db()[source]

Get the database connection for the current request.

Uses Flask’s g object to cache the connection per-request. The connection uses sqlite3.Row as row factory so that columns can be accessed by name.

Returns:

The active database connection.

Return type:

sqlite3.Connection

models.close_db(e=None)[source]

Close the database connection at the end of the request.

Parameters:

e – An optional exception passed by Flask’s teardown mechanism.

models.init_db(app)[source]

Initialize the database and register the teardown hook.

Creates the posts table if it does not exist and registers close_db() as a teardown handler.

Parameters:

app (Flask) – The Flask application instance.

models.get_all_posts(category=None, page=None, per_page=5)[source]

Retrieve posts with optional category filter and pagination.

When page is None, returns a plain list of all matching posts (backwards-compatible). When page is an integer, returns a dict containing the paginated posts and metadata.

Parameters:

  • category (str, optional) – Filter by category name.

  • page (int, optional) – The 1-based page number.

  • per_page (int) – Number of posts per page. Defaults to 5.

Returns:

A list of sqlite3.Row when page is None,

or a dict with keys posts, page, total_pages, and total when paginating.

Return type:

list or dict

models.search_posts(query)[source]

Search posts by title or content using SQL LIKE.

Parameters:

query (str) – The search keyword.

Returns:

A list of sqlite3.Row matching the query,

ordered by creation date descending.

Return type:

list
